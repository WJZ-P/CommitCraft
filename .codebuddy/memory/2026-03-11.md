# 2026-03-11

## 修复：前端 Zpix 字体加载报错 "Unsupported OpenType signature wOF2"
- **原因**：`opentype.js` 不支持解析 `.woff2` 格式。
- **修复**：统一使用 `.ttf` 格式。
  - 删除 `public/fonts/zpix.woff2`，下载 `public/fonts/zpix.ttf`（~6.85MB）。
  - CSS `@font-face` 改为 `url('/fonts/zpix.ttf') format('truetype')`。
  - 前端三个组件（RepoCard、ProfileCard、BannerHall）加载地址改为本地 `/fonts/zpix.ttf`。
  - 服务端 `fontBaker.ts` 仍用 CDN ttf（服务端无法直接访问 /public 静态路径）。

## 搭建 next-intl 多语言框架（基础骨架）
- **方案**：使用 `next-intl`（无 i18n 路由模式），通过 Cookie 切换语言，不改变 URL 结构。
- **新增文件**：
  - `messages/en.json` / `messages/zh.json`：中英文翻译文件，已提取 page.tsx 中所有用户可见文本。
  - `i18n/request.ts`：next-intl 请求配置，从 Cookie 读取 locale。
- **修改文件**：
  - `next.config.mjs`：引入 `createNextIntlPlugin`，用 `withNextIntl()` 包裹配置。
  - `app/layout.tsx`：改为 async，添加 `NextIntlClientProvider` 包裹 children，动态设置 `<html lang>`。
  - `app/page.tsx`：引入 `useTranslations` / `useLocale`，导航栏新增语言切换按钮（Cookie + reload）。
- **状态**：框架已搭好，`t()` 翻译函数已可用但尚未替换页面中的硬编码文本。
